## 核心判断

Geometry Board 不是把内容装进一个漂亮模板，而是用少量几何元素把关系画出来。点、线、面、体是视觉语言的基本单位；高级感来自比例、对齐、留白、线条层级、尺度和克制的强调色，不来自装饰数量。

画板只承担一个核心判断：让读者一眼看懂谁与谁有关、关系如何发生、重点在哪里。背景、证据、例子和完整解释留在正文中。

## 点线面体的生成关系

点、线、面、体不是四种孤立的图标，而是一条从位置到系统的生成链：

| 元素 | 几何含义 | 默认表达 | 生成方式 |
|---|---|---|---|
| 点 | 位置，没有宽度和长度 | 对象、角色、事件、信号、状态 | 一个位置或一个可识别的节点 |
| 线 | 点的移动或两个点的连接 | 路径、方向、依赖、流动、边界 | 连接、延伸、转折、重复 |
| 面 | 有宽度的线，或被边界围住的区域 | 群组、阶段、范围、环境、共同属性 | 封闭、铺开、叠加、重复 |
| 体 | 有高度、宽度和深度的空间对象 | 系统、层级、空间剖面、整体结构 | 面的叠层、透视、轴测、交叠 |

从点到体时，不要为了“立体感”增加真实光影。优先使用轴测关系、层叠、透明度、切面、边界和空间距离，让体积服务于结构。

## 关系如何被看见

同一批元素可以通过不同视觉变量表达不同关系。每次只给变量一个主要含义：

| 视觉变量 | 优先表达 |
|---|---|
| 位置 | 分类、时间、坐标、优先级、空间位置 |
| 距离 | 关系强弱、隔离、阶段间断、相似程度 |
| 尺度 | 重要程度、影响范围、数量或层级 |
| 方向 | 流向、因果、行动顺序、张力 |
| 曲率 | 流动、反馈、回路、绕行和跨空间连接 |
| 线宽 | 关系强度、主路径与次路径 |
| 对齐 | 共同属性、共同基线、同一层级 |
| 重复 | 频率、节奏、规模、规则 |
| 包含 | 群组、范围、上层与下层、约束 |
| 重叠 | 交集、协同、冲突、共享空间 |
| 透明度 | 前后关系、叠加、不确定性、弱关系 |
| 空白 | 权重、独立性、阅读停顿、未连接区域 |

不要同时用位置、颜色、大小、线宽和曲率表达同一件事，除非需要强化重点。曲率优先只表达“关系如何流动”；颜色默认只表达“当前重点”，不承担完整分类系统。

## 基本设计原则

### 节奏与平衡

重复产生节奏，变化产生重点。相等间距带来秩序，逐渐变化的间距带来方向；对称提供稳定，不对称可以产生张力。先建立可读的平衡，再引入变化，不用随机摆放制造所谓的高级感。

弧线也必须有秩序：单条弧线服从起点、终点和关系方向；多条弧线共享圆心、控制点或曲率，并用等距偏移、对称或镜面对称形成节奏。自然不等于随意，曲线的柔和感必须建立在几何约束上。

### 尺度与层级

大小差异要有理由：表达重要性、范围、数量或空间层级。不要让所有对象同样大，也不要让尺寸变化没有解释。主对象、主路径和辅助信息应有清楚的重量差。

### 格式塔与阅读

接近产生分组，相似产生归类，连续产生路径，闭合产生整体，前景—背景产生重点。先利用位置和关系让读者看懂，再用文字补充，不用长解释弥补错误的分组。

### 边界、图层与透明度

边界只在确实需要表达范围、模块或比较时使用。图层用于区分阶段或系统层级，透明度用于表达交叠和前后关系。若边框遮住主路径，删除边框，改用点、线、面、空白或空间距离。

### 模块、网格与模式

模块适合重复出现的能力单元、角色单元或流程单元；网格负责对齐、比例和阅读秩序；模式适合表达重复规则、群体密度和连续性。不要把网格画成装饰，也不要把每个模块做成同样大小的卡片。

### 图示、时间与规则

图示要把关系变成可追踪的路径。时间用轴线、位置、阶段面或连续变化表达；动态感用方向、重复、间距变化和反馈线表达。随机性只能在明确规则下使用，例如点阵扰动或路径变化，不能让布局失去阅读顺序。

### 色彩与 Geometry Blue

基础色保持白底、黑白灰。`#2F6BFF` 是默认 Geometry Blue，只用于核心对象、主路径、当前状态、风险或变化部分；面积原则上低于 8%。高级蓝色的关键不是更亮或更复杂，而是面积小、位置准、与灰阶形成清楚对比。

蓝色圆点是可选的状态定位点。它可以标记新信息、反馈、决策、信号或当前焦点，但不能为了展示圆点而展示圆点。启用前必须能回答两个问题：这个点发生了什么？它改变了哪条关系？圆点优先落在几何中心、主轴、转折点或连接端点，并与主路径真实相连。多个蓝色圆点只有在表达同一类状态或可比较事件时才成立，数量不设固定上限，但必须服从语义和布局密度；删掉后若关系仍然完整，就不要使用。

## 从内容到画板

按以下顺序工作：

1. 用一句直白的话写出读者必须看懂的核心判断。
2. 区分对象和关系：对象需要边界，关系需要路径。
3. 判断使用卡片、点线或混合；不要先套模板。
4. 为对象选择点、圆、矩形、面或体，为关系选择直线、自然弧线、方向、距离、重叠或透明度；轴线和边界用直线，流动和反馈优先用弧线。
5. 只选择一到两个主要视觉变量表达关系，其余变量保持克制。
6. 建立主轴、共同基线或几何中心，再安排紧密度和留白。
7. 用 Geometry Blue 标出重点；蓝色圆点仅在能定位变化、反馈、决策或当前焦点时启用，删掉无法说明语义的装饰。
8. 缩小到 25% 检查重点、方向、层级和主关系是否仍然清楚。

最终检查：如果去掉文字后仍能看出主要结构，说明几何关系成立；如果只剩一排盒子或一堆线，说明还需要重新判断表达方式。

## 画布与网格

| 预设 | 比例 | 默认用途 |
|---|---:|---|
| 横向 | 16:9 | 汇报、方案、屏幕阅读 |
| 标准 | 4:3 | 文档内嵌、结构说明 |
| 纵向 | A4 | 报告、长文档 |

默认画布为 `1200 × 675 px`。使用 8 px 基础网格；外边距约为画布宽度的 8%，安全区 64–96 px；元素间距优先使用 8 / 16 / 24 / 40 / 64 px。对齐网格或明确的几何轴线，边缘保留呼吸感。画面应像一张正在编辑的结构图，有清楚的主轴、定位关系或几何中心。

## 颜色、线条与字体

基础色：

| 角色 | 色值 |
|---|---|
| 主文字 | `#111111` |
| 次级文字 | `#666666` |
| 辅助文字 | `#999999` |
| 主线条 | `#222222` |
| 辅助线条 | `#B8B8B8` |
| 分隔线 | `#E8E8E8` |
| 浅灰填充 | `#F5F5F5` |
| 背景 | `#FFFFFF` |
| 默认强调色 Geometry Blue | `#2F6BFF` |

每张图最多增加一个强调色，面积原则上低于画布 8%，只用于核心对象、关键路径、风险、当前状态或变化部分。

线条默认使用 0.5 px 辅助线、1 px 主结构线、1.5 px 强调线。虚线表达未来状态、弱关系或边界；连线优先直线、折线或轻微曲线。箭头小于正文高度，禁止粗大箭头。

中文字体优先苹方、思源黑体、HarmonyOS Sans 或系统无衬线；英文与数字优先 Inter、SF Pro、Helvetica Neue 或 IBM Plex Sans。主标题 28–36 px，核心结论 18–24 px，节点标签 13–16 px，注释 10–12 px；整图不超过三种文字层级。

## 文字密度

默认采用“直白易懂、少字模式”：标题 + 关键词节点 + 必要关系词即可。优先使用日常业务语言和具体动作，让陌生读者只看画板也能说出谁、做什么、结果是什么。中文可见文字默认不超过 80 字，复杂结构硬上限 120 字；副标题尽量不超过 18 个汉字，节点尽量控制在 2–8 字。

把完整解释放进文档正文，不要同时堆叠标题、长副标题、阶段说明、节点说明和 footer。优先写“先说清楚”“放回原文”这类具体动作，少写没有动作主体的抽象名词。文字变多时，优先删除背景、例子、重复判断和限定句，再考虑拆图；不要通过缩小字号、压缩间距或填满留白来容纳更多字。

## 图形语义

| 图形 | 默认语义 |
|---|---|
| 点 | 个体、信号、事件、节点 |
| 线 | 联系、流动、依赖、路径 |
| 圆 | 主体、循环、系统核心 |
| 矩形 | 模块、能力、功能范围 |
| 平面 | 层级、基础、环境 |
| 立方体 | 完整系统、组织单元 |
| 圆柱体 | 数据、资源、存储 |
| 点阵 | 群体、规模、分布 |
| 虚线框 | 潜在范围、未来状态 |
| 重叠区域 | 交集、协同、冲突 |
| 空白距离 | 权重、隔离、关系强弱 |

同一批图示中保持图形语义稳定。每个元素都要回答“它表达了什么关系”；答不上来就删除。

## 表达方式选择

先判断读者要看的是对象本身，还是对象之间的关系：

| 表达方式 | 适用情况 | 主要风险 |
|---|---|---|
| 卡片 | 独立模块、并列比较、明确边界、每项有自己的短说明 | 变成等尺寸卡片墙，关系被边框遮住 |
| 点与线 | 顺序、路径、依赖、网络、流动、信号、变化、共同中心 | 线太多、方向不清、节点失去层级 |
| 混合 | 模块边界真实存在，同时模块之间有重要路径或依赖 | 卡片和连线各说各话，主关系被容器打断 |

判断规则：去掉边框后关系仍然清楚，就优先使用点线；去掉边界后独立模块无法识别，才保留卡片。卡片不是禁用项，卡片墙才是问题。卡片内默认只放标题和一条短说明，只有边界、比较或模块属性确实需要时才使用。

## 版式与布局

- **居中**：主视觉中心落在画布主轴或有意设置的视觉中心；文字用明确的中心锚点和统一基线，避免几何中心、文字中心和视觉重心互相打架。
- **对齐**：使用 8 px 网格、共同基线、共同轴线和统一内边距；不接受“差一点对齐”的边缘错位。
- **排列**：默认左到右、上到下；一张图只保留一条主路径，相关元素成组，不相关元素拉开，连线尽量不交叉。
- **紧密度**：同组元素使用 8 / 16 / 24 px，相关组之间使用 40 / 64 px，独立组之间保留更大空白；距离表达关系强弱，不把元素平均铺满画布。
- **密度**：关系密集处可以紧凑，但必须有清晰层级；关系稀疏处保留空白，不用组件填空。留白是结构的一部分。
- **重点**：主对象、主路径和次要信息通过尺寸、线宽、对比度或颜色分级，禁止多个同等强度的视觉中心。

## 几何编辑感与卡片感边界

几何编辑感不是增加装饰线，而是让结构本身可见：主轴负责方向，辅助线负责对齐，点负责定位，面和层负责范围，交叠负责关系，空白负责权重。

避免以下视觉结果：三张或多张等尺寸卡片并列、每个节点都放进圆角矩形、胶囊标签堆叠、统一阴影、仪表盘式模块墙、每块内容都有同等视觉重量。若信息关系被卡片边框遮蔽，应删除容器，改用线、点、层、空间或一个中心几何体重排。

高级审美检查：一个主焦点、一个主路径、两到三种线宽、黑白灰加一个强调色、足够留白；缩小到 25% 时仍能一眼看出重点、方向和层级。

## 八种构图模板

| ID | 适合表达 | 结构要点 |
|---|---|---|
| `axis-flow` | 时间、流程、因果、演进、任务执行 | 单一主轴，节点依次排列，次要信息置于轴线两侧 |
| `layered-architecture` | 技术/产品/组织/基础设施分层 | 3–5 层，上层依赖下层，允许平面叠加或剖面表达 |
| `radial-center` | 核心平台、中心角色、生态、多方协同 | 一个核心，3–8 个外围对象，控制连线密度 |
| `matrix-2d` | 分类、优先级、成熟度、价值成本、风险影响 | 两个明确坐标轴，对象用点或小几何体表达 |
| `input-process-output` | AI 能力、系统机制、数据加工、业务闭环 | 左中右三区，中间模块为重点，可带反馈回路 |
| `dot-filter` | 人才筛选、群体分类、转化、规模变化、聚类 | 点阵表示总体，颜色/密度/位置表达差异 |
| `section-space` | 内部结构、组织层次、平台能力、多层运行机制 | 建筑制图式正投影或等轴测，不用真实光影 |
| `tension-contrast` | 冲突、权衡、竞争、旧新模式、目标现实 | 2–3 个核心几何体，用距离、方向、重叠和比例表达 |

构图选择以关系为准：顺序选轴线，依赖选分层，中心多方选辐射，二维评价选矩阵，加工链选输入—处理—输出，群体收敛选点阵，内部层次选切面，冲突对比选张力。

## 视觉审查

按以下权重进行内部评分：核心观点清晰度 25%，信息关系准确度 20%，构图与平衡 15%，留白与密度 10%，字体与层级 10%，线条与对齐 10%，色彩克制 5%，风格一致性 5%。目标分数不低于 80。

触发重排或删减的条件：元素重叠、文字溢出、读者需要术语表才能理解、出现三个以上等尺寸信息卡片、去掉文字后只剩盒子、留白低于 30%、交叉连线超过 3 条、节点超过 18 个、中文可见文字超过 80 字（复杂结构超过 120 字）、同时出现五层文案、两个以上同等强度视觉中心、颜色超过黑白灰加一个强调色，或缩小至 25% 后无法理解整体结构。

优先修复顺序：删掉次要信息 → 压缩长句为关键词 → 删除重复副标题/footer → 调整构图 → 重排节点与留白 → 统一字号/线宽 → 收紧颜色。不要通过增加装饰或缩小字号来掩盖文字过多。
